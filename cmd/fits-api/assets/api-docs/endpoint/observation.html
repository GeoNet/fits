Observation

Observations as CSV

Method: GET
URI
/observation?typeID=(typeID)&siteID=(siteID)&[days=int]&[methodID=(methodID)]
Accept
text/csv;version=1

Query Parameters

Required:
siteID
Site identifier e.g., WI000.
typeID
A type identifier for observations e.g., e.
Optional:
days
The number of days of data to select before now e.g., 250. Maximum value is 365000.
methodID
A valid method identifier for observation type e.g., doas-s. typeID must be specified as well.

Response Properties

column 1
The date-time of the observation in ISO8601 format, UTC time zone.
column 2
The observation value.
column 3
The observation error. 0 is used for an unknown error.

Spatial Observation

Spatial observations as CSV

Method: GET
URI
/observation?typeID=(typeID)&start=(ISO8601 date time)&days=(int)&[srsName=(CRS)]&[within=POLYGON((...))]&[methodID=(methodID)]
Accept
text/csv;version=1

Query Parameters

Required:
days
The number of days of data to select from the start e.g., 1. Range is 1-7.
start
the date time in ISO8601 format for the start of the time window for the request e.g., 2014-01-08T12:00:00Z.
typeID
A type identifier for observations e.g., e.
Optional:
methodID
A valid method identifier for observation type e.g., doas-s.
srsName
Default EPSG:4326. Specify the spatial reference system to project site coordinates to e.g., EPSG:27200 (New Zealand Map Grid). Site locations are stored as a geography. For projection they are cast to a geometry and then projected using ST_Transform. Behaviour of projection is not defined outside the bounds of the SRS. For further details please refer to the PostGis manual: 4.3.1. The SPATIAL_REF_SYS Table and Spatial Reference Systems.
within
Only return sites that fall within the polygon (uses ST_Within). The polygon is defined in WKT format (WGS84). The polygon must be topologically closed. Spaces can be replaced with + or URL encoded as %20 e.g., POLYGON((177.18+-37.52,177.19+-37.52,177.20+-37.53,177.18+-37.52)).

Response Properties

column 1
Site identifier e.g., WI000.
column 2
The longitude (X) of the observation site.
column 3
The latitude (Y) of the observation site.
column 4
Site height (m).
column 5
Site ground relationship (m). Sites above ground level have a negative ground relationship.
column 6
The date-time of the observation in ISO8601 format, UTC time zone.
column 7
The observation value.
column 8
The observation error. 0 is used for an unknown error.
